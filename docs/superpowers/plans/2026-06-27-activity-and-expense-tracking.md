# Module 2 (Activity & Expense Tracking) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Module 2 — record farm activities (e.g. fertilizing, spraying) and expenses (direct per crop cycle, or farm-wide overhead) so true cost per crop cycle can be computed.

**Architecture:** Laravel 13 + Inertia v3 (React 19). Master data `activity_types` and `expense_categories` get simple inline-CRUD management pages (same pattern as fruit-types/varieties). Transactional `activities` belong to a `crop_cycle`; `expenses` belong to an `expense_category` and optionally to a `crop_cycle` (set = direct cost) and an `activity` (cost incurred by an activity). A new crop-cycle detail page (`crop-cycles/show`) is where activities and direct expenses are recorded and the running direct-cost total is shown. Overhead expenses (`crop_cycle_id = null`) are managed on a dedicated farm-wide expenses page. Pest feature tests with `RefreshDatabase` cover every endpoint.

**Tech Stack:** PHP 8.5, Laravel 13, Inertia v3, React 19, TypeScript, Tailwind v4, Wayfinder, Pest 4.

**Scope note:** This is plan 2 of 4 (data-model spec `docs/superpowers/specs/2026-06-27-farm-erp-data-model-design.md`). Tables in scope: `activity_types`, `expense_categories`, `activities`, `expenses`. Module 1 (plan 1) is complete: `users.role`, `fruit_types`, `fruit_varieties`, `plots`, `crop_cycles` all exist. Modules 3 (harvest/sales) and 4 (analytics) follow in later plans. Master tables `grades` ships with module 3.

**Conventions to follow (study before starting):**
- Models use `#[Fillable([...])]` attribute + `@property` docblocks (see `app/Models/CropCycle.php`).
- Enums use TitleCase keys with a `label()` method returning Thai (see `app/Enums/CropCycleStage.php`).
- Controllers return `Inertia::render` for pages, `to_route(...)` for redirects, and flash with `Inertia::flash('toast', ['type' => 'success', 'message' => '...'])` (see `app/Http/Controllers/Farm/FruitTypeController.php`, `CropCycleController.php`).
- FormRequests: `authorize(): bool` returns `true`, `rules(): array` typed `array<string, mixed>` (see `app/Http/Requests/Farm/StoreCropCycleRequest.php`).
- Feature tests: `php artisan make:test --pest Farm/<Name>`, run with `php artisan test --compact --filter=<Name>`. Every controller test starts with a `guests cannot ...` redirect test.
- Inertia pages: `<Form {...Controller.action.form(id)}>` render-prop pattern, `setLayoutProps({ breadcrumbs: [...] })` inside the component for dynamic breadcrumbs (see `resources/js/pages/farm/plots/show.tsx`). Inline-CRUD pattern with `useState` editingId for master pages (see `resources/js/pages/farm/fruit-types/index.tsx`).
- All user-facing strings (UI + validation messages + toasts) are **Thai**. Code identifiers stay English.
- After any PHP change, run `vendor/bin/pint --dirty --format agent`.
- After adding/changing routes or controllers, run `php artisan wayfinder:generate` so `@/actions/**` and `@/routes/**` imports resolve. `resources/js/actions` and `resources/js/routes` are gitignored — they are regenerated by the Vite plugin, so do not `git add` them.
- Shared TS types live in `resources/js/types/farm.ts` — extend, don't duplicate.

```php
<?php

namespace App\Models;

use App\Enums\ExpenseScope;
use Database\Factories\ExpenseFactory;
use Illuminate\Database\Eloquent\Attributes\Fillable;
use Illuminate\Database\Eloquent\Builder;
use Illuminate\Database\Eloquent\Casts\Attribute;
use Illuminate\Database\Eloquent\Factories\HasFactory;
use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Relations\BelongsTo;
use Illuminate\Support\Carbon;

/**
 * @property int $id
 * @property int $expense_category_id
 * @property string $amount
 * @property Carbon $spent_on
 * @property string|null $description
 * @property int|null $crop_cycle_id
 * @property int|null $activity_id
 * @property int $recorded_by
 * @property ExpenseScope $scope
 * @property Carbon|null $created_at
 * @property Carbon|null $updated_at
 */
#[Fillable(['expense_category_id', 'amount', 'spent_on', 'description', 'crop_cycle_id', 'activity_id', 'recorded_by'])]
class Expense extends Model
{
    /** @use HasFactory<ExpenseFactory> */
    use HasFactory;

    protected function casts(): array
    {
        return [
            'amount' => 'decimal:2',
            'spent_on' => 'date',
        ];
    }

    /**
     * @return BelongsTo<ExpenseCategory, $this>
     */
    public function expenseCategory(): BelongsTo
    {
        return $this->belongsTo(ExpenseCategory::class);
    }

    /**
     * @return BelongsTo<CropCycle, $this>
     */
    public function cropCycle(): BelongsTo
    {
        return $this->belongsTo(CropCycle::class);
    }

    /**
     * @return BelongsTo<Activity, $this>
     */
    public function activity(): BelongsTo
    {
        return $this->belongsTo(Activity::class);
    }

    /**
     * @return BelongsTo<User, $this>
     */
    public function recordedBy(): BelongsTo
    {
        return $this->belongsTo(User::class, 'recorded_by');
    }

    /**
     * @return Attribute<ExpenseScope, never>
     */
    protected function scope(): Attribute
    {
        return Attribute::get(
            fn (): ExpenseScope => $this->crop_cycle_id === null
                ? ExpenseScope::Overhead
                : ExpenseScope::Direct
        );
    }

    /**
     * @param  Builder<Expense>  $query
     */
    public function scopeDirect(Builder $query): void
    {
        $query->whereNotNull('crop_cycle_id');
    }

    /**
     * @param  Builder<Expense>  $query
     */
    public function scopeOverhead(Builder $query): void
    {
        $query->whereNull('crop_cycle_id');
    }
}
```

Note: the `scope()` accessor exposes the `scope` attribute; the `scopeDirect`/`scopeOverhead` methods are Eloquent local query scopes (`Expense::direct()`, `Expense::overhead()`). These do not collide — local scopes are matched by the `scope` prefix + method name.

```php
<?php

namespace App\Http\Controllers\Farm;

use App\Http\Controllers\Controller;
use App\Http\Requests\Farm\StoreActivityRequest;
use App\Models\Activity;
use App\Models\CropCycle;
use Illuminate\Http\RedirectResponse;
use Inertia\Inertia;

class ActivityController extends Controller
{
    public function store(StoreActivityRequest $request, CropCycle $cropCycle): RedirectResponse
    {
        $validated = $request->validated();

        $activity = $cropCycle->activities()->create([
            'activity_type_id' => $validated['activity_type_id'],
            'performed_on' => $validated['performed_on'],
            'notes' => $validated['notes'] ?? null,
            'recorded_by' => $request->user()->id,
        ]);

        if (! empty($validated['cost'])) {
            $activity->expenses()->create([
                'expense_category_id' => $validated['expense_category_id'],
                'amount' => $validated['cost'],
                'spent_on' => $validated['performed_on'],
                'crop_cycle_id' => $cropCycle->id,
                'recorded_by' => $request->user()->id,
            ]);
        }

        Inertia::flash('toast', ['type' => 'success', 'message' => 'บันทึกกิจกรรมแล้ว']);

        return to_route('crop-cycles.show', $cropCycle);
    }

    public function destroy(Activity $activity): RedirectResponse
    {
        $cropCycleId = $activity->crop_cycle_id;
        $activity->delete();

        Inertia::flash('toast', ['type' => 'success', 'message' => 'ลบกิจกรรมแล้ว']);

        return to_route('crop-cycles.show', $cropCycleId);
    }
}
```

```php
use Illuminate\Foundation\Http\FormRequest;

class StoreExpenseRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true;
    }

    /**
     * @return array<string, mixed>
     */
    public function rules(): array
    {
        return [
            'expense_category_id' => ['required', 'exists:expense_categories,id'],
            'amount' => ['required', 'numeric', 'min:0.01'],
            'spent_on' => ['required', 'date'],
            'description' => ['nullable', 'string', 'max:255'],
        ];
    }
}
```

- [ ] **Step 4: Create the controller**

`app/Http/Controllers/Farm/ExpenseController.php`:

```php
<?php

namespace App\Http\Controllers\Farm;

use App\Http\Controllers\Controller;
use App\Http\Requests\Farm\StoreExpenseRequest;
use App\Models\CropCycle;
use App\Models\Expense;
use App\Models\ExpenseCategory;
use Illuminate\Http\RedirectResponse;
use Inertia\Inertia;
use Inertia\Response;

class ExpenseController extends Controller
{
    public function index(): Response
    {
        return Inertia::render('farm/expenses/index', [
            'expenses' => Expense::overhead()
                ->with('expenseCategory')
                ->latest('spent_on')
                ->get(),
            'expenseCategories' => ExpenseCategory::orderBy('name')->get(['id', 'name']),
        ]);
    }

    public function store(StoreExpenseRequest $request): RedirectResponse
    {
        Expense::create([
            ...$request->validated(),
            'crop_cycle_id' => null,
            'recorded_by' => $request->user()->id,
        ]);

        Inertia::flash('toast', ['type' => 'success', 'message' => 'บันทึกค่าใช้จ่ายส่วนกลางแล้ว']);

        return to_route('expenses.index');
    }

    public function storeForCycle(StoreExpenseRequest $request, CropCycle $cropCycle): RedirectResponse
    {
        $cropCycle->expenses()->create([
            ...$request->validated(),
            'recorded_by' => $request->user()->id,
        ]);

        Inertia::flash('toast', ['type' => 'success', 'message' => 'บันทึกค่าใช้จ่ายแล้ว']);

        return to_route('crop-cycles.show', $cropCycle);
    }

    public function destroy(Expense $expense): RedirectResponse
    {
        $cropCycleId = $expense->crop_cycle_id;
        $expense->delete();

        Inertia::flash('toast', ['type' => 'success', 'message' => 'ลบค่าใช้จ่ายแล้ว']);

        if ($cropCycleId !== null) {
            return to_route('crop-cycles.show', $cropCycleId);
        }

        return to_route('expenses.index');
    }
}
```

## Notes for the implementer

- **Direct vs overhead:** an expense is **direct** when `crop_cycle_id` is set, **overhead** when null. The `Expense::direct()` / `Expense::overhead()` query scopes and the `scope` accessor enforce this. Never store a `scope` column — it is derived.
- **Activity → expense link:** recording an activity with a `cost` creates one `Expense` with both `activity_id` and `crop_cycle_id` set. Deleting the activity cascade-deletes that expense.
- **Wayfinder imports:** `@/actions/App/Http/Controllers/Farm/<Controller>` exposes named action helpers (`store`, `update`, `destroy`) with `.form()`/`.url()`. `@/routes/<resource>` exposes `index()`, `show(id)`, etc. Always run `php artisan wayfinder:generate` after route changes before building. Do not `git add resources/js/actions` or `resources/js/routes` (gitignored).
- **Thai everywhere:** all visible strings, validation messages, and toasts are Thai; identifiers stay English.
- **`amount` serializes as a string** (`decimal:2` cast) — the frontend uses `Number(expense.amount)` for arithmetic and `.toLocaleString('th-TH', ...)` for display.
- **`InputError` is a default export** (`import InputError from '@/components/input-error';`) — follow the sibling pages, not a named import.
